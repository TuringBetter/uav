# 核心目标
基于以下需求，为面向弱网无人系统的分布式协同协议开发代码框架与核心实现，核心优化方向是降低通信开销，并适配弱网（丢包、延迟、带宽受限）+ FANET 拓扑场景，重点实现三层解耦架构的节点通信程序，最终可通过 ns-3 仿真验证性能优势。
# 一、项目背景与核心要求
- 毕设方向：面向弱网无人系统的分布式协同协议设计，传输对象为时序数据
- 核心目标：改进现有一致性算法 / 机制，显著降低通信开销，并通过弱网实验对比证明优于主流方案
- 实验平台演进：
阶段 1：单机多节点通信模拟（Go 编写节点 + Docker 模拟节点 + TCP/UDP 通信）
阶段 2：引入弱网模拟（丢包、延迟、带宽限制）
阶段 3：接入 ns-3 + FANET 拓扑模型
阶段 4：集成可替换的分布式共识算法
# 二、架构设计
1. 三层解耦架构（核心约束）
```
┌───────────────────────┐
│   分布式算法层          │  ← 可替换（算法无关）
├───────────────────────┤
│   节点通信框架层        │  ← 通用（核心解耦层）
├───────────────────────┤
│   网络传输层            │  ← 通用（仅负责收发）
└───────────────────────┘
```
2. 各层职责与模块划分

|层级|核心职责|子模块/文件结构|
|--|--|--|
|网络传输层|仅负责消息发送/接收，不感知算法；提供统一收发接口|node/transport/：<br> - tcp/（TCP 实现）<br> - udp/（UDP 实现）<br> - codec/（编解码）|
|节点通信框架层|解耦算法与网络；负责节点生命周期、消息路由、定时器、Peer 管理|node/runtime/：<br> - node.go（节点主体）<br> - router.go（消息分发）<br> - timer.go（定时器）<br> - peer.go（节点列表管理）|
|分布式算法层|可替换；仅依赖框架层接口实现业务逻辑|node/algorithm/：<br> - centralized/（中心化协调）<br> - gossip/（Gossip）<br> - raft/（Raft）<br> - weaknet/（自定义弱网算法）|
|配置层|统一管理节点、网络、算法配置|node/config/|
3. 关键接口设计
（1）算法层接口（框架层向算法层暴露）
```go
// 算法层必须实现此接口
type Algorithm interface {
    Start()          // 算法启动
    Stop()           // 算法停止
    OnMessage(msg Message)  // 框架层转发网络消息时调用
    OnTick()         // 定时器触发时调用
}
```
（2）框架层核心能力（算法层调用）
```go
// 算法层通过框架层提供的以下接口收发消息/管理定时器，不直接操作网络
node.Send(peerID uint16, msg Message)       // 单播
node.Broadcast(msg Message)                 // 广播
node.SetTimer(duration time.Duration)       // 设置定时器（触发OnTick）
```
（3）框架层自动处理逻辑
- 网络层收到消息 → 路由分发 → 调用算法层 OnMessage
- 定时器触发 → 调用算法层 OnTick
# 三、Message 结构体定义（强制遵循）
需适配弱网特性（去重、乱序恢复、过期丢弃、优先级调度），结构如下：
```go
// 固定头部 + 算法自定义Payload（二进制编码）
type Message struct {
    Type       uint8   // 消息类型（HEARTBEAT/STATE/CONSENSUS等，用于路由）
    From       uint16  // 发送节点ID（去中心化邻居判断）
    To         uint16  // 目标节点ID（0=广播）
    Seq        uint32  // 递增序列号（去重/丢包检测）
    Timestamp  int64   // 发送时间戳（毫秒级，延迟计算/过期判断）
    TTL        uint16  // 有效期（ms，弱网过期丢弃）
    Priority   uint8   // 优先级（0=最高，弱网队列调度）
    Payload    []byte  // 算法自定义内容（二进制，由算法层编解码）
}
```
Message 字段约束

|字段|取值 / 用途|
|--|--|
|Type|枚举值：HEARTBEAT (0)/STATE (1)/CONTROL (2)/CONSENSUS (3)/CUSTOM (4)|
|To|	单播 = 目标节点 ID；广播 = 0；组播 = 自定义组 ID|
|Seq|	每个节点独立递增，用于弱网下去重、乱序恢复|
|TTL|	避碰信息 = 100ms；状态信息 = 500ms；其他可配置|
|Priority|	0 = 避碰（最高）/1 = 状态 / 2 = 任务 / 3 = 调试|

# 四、开发优先级与核心优化点
1. 优先实现网络传输层 + 框架层的通用能力（保证解耦性）；
2. 算法层先实现基础版 Raft/Gossip，再开发自定义弱网算法（重点优化通信开销）；
3. 弱网适配关键优化：
- 基于 Seq 实现消息去重 / 乱序恢复；
- 基于 TTL 自动丢弃过期消息；
- 基于 Priority 实现发送队列优先级调度；
- 时序数据传输时压缩 Payload 降低带宽占用。
